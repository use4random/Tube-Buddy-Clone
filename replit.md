# TubePulse

A YouTube creator optimization platform that helps creators grow their channels with SEO tools, keyword research, A/B testing, bulk editing, analytics, and AI-powered content tools.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 5000)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5 (port 8080 internally, proxied to /api)
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)
- Frontend: React + Vite + Tailwind + shadcn/ui + Recharts + wouter + TanStack Query

## Where things live

- `lib/api-spec/openapi.yaml` — source of truth for all API contracts
- `lib/api-client-react/src/generated/` — generated hooks and Zod schemas (do not edit)
- `lib/api-zod/src/` — server-side Zod schemas generated by Orval
- `lib/db/src/schema/` — Drizzle ORM table definitions (9 schema files)
- `artifacts/api-server/src/routes/` — all Express route handlers
- `artifacts/api-server/src/lib/auth.ts` — JWT + bcrypt helpers
- `artifacts/api-server/src/middlewares/requireAuth.ts` — auth middleware
- `artifacts/web/src/pages/` — all frontend pages
- `artifacts/web/src/components/layout/` — sidebar + dashboard layout
- `artifacts/web/src/lib/auth.tsx` — AuthContext (JWT stored in localStorage)
- `artifacts/web/src/lib/channel.tsx` — ChannelContext (selected channel)

## Architecture decisions

- Contract-first: OpenAPI spec → codegen → hooks (client) and Zod schemas (server). Never hand-write API types.
- Auth: bcryptjs + jsonwebtoken stored in localStorage (no session cookies). Bearer token sent in Authorization header.
- All protected routes check `requireAuth` middleware which validates JWT.
- Billing is hardcoded (no Stripe integration yet). Plans are served from `/api/billing/plans`.
- Analytics and keyword data are simulated (no real YouTube API integration yet).
- The `query: { enabled: bool } as any` pattern is used in React hooks because orval generates `UseQueryOptions` which requires `queryKey` in TanStack Query v5, but the key is auto-generated by the hook.

## Product

- **Dashboard** — views/subscriber/revenue overview + daily chart + top videos
- **Keyword Research** — search any term, see volume/competition/score + trend chart + related keywords
- **SEO Studio** — score title/description/tags in real time with actionable recommendations
- **A/B Testing** — create title or thumbnail experiments, apply winners with confidence scores
- **Bulk Editor** — bulk update or find/replace across multiple videos
- **Analytics** — channel metrics over 7d/30d/90d/1y + heatmap of best publish times
- **Competitors** — track and compare competing channels (subs, avg views, upload frequency)
- **Comments** — view/reply to comments with canned responses + quick-reply picker
- **AI Tools** — AI title generator, description writer, tag suggester, video idea planner
- **Billing** — plan comparison (Free $0 / Pro $4.50 / Legend $28.99 / Enterprise custom)

## Gotchas

- Always run `pnpm --filter @workspace/api-spec run codegen` after editing `lib/api-spec/openapi.yaml`
- Run `pnpm --filter @workspace/db run push` after editing DB schema files
- Do NOT use `useGetBillingPlans` — generated hook is `useListBillingPlans`
- The `query: { enabled: bool }` options need ` as any` casts due to TanStack Query v5 + orval type incompatibility
- API server builds with esbuild (not tsc) — typecheck separately with `pnpm --filter @workspace/api-server run typecheck`
