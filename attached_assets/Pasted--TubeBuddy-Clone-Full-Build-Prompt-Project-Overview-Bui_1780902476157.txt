# TubeBuddy Clone — Full Build Prompt

## Project Overview

Build a full-stack YouTube creator optimization platform that replicates and extends TubeBuddy's functionality. The product is a **browser extension + web dashboard SaaS** that integrates deeply with YouTube Studio to provide SEO optimization, keyword research, A/B testing, bulk editing, analytics, comment management, and AI-powered content tools.

The platform will be monetized via tiered subscriptions (Free, Pro, Legend, Enterprise) and must integrate with the YouTube Data API v3, Stripe for billing, and the Anthropic Claude API for AI features.

---

## Product Requirements

### What TubeBuddy Does (Competitor Reference)

TubeBuddy is a browser extension + SaaS platform for YouTube creators. It injects tools directly into YouTube Studio and provides a standalone web dashboard. Key value propositions:

- YouTube SEO optimization (keyword research, tag suggestions, video scoring)
- A/B testing of thumbnails and titles to maximize click-through rate
- Bulk editing of video metadata across an entire channel library
- Comment management and moderation tools
- Analytics and competitor channel tracking
- AI-generated titles, descriptions, and video ideas
- Multi-channel management for agencies and enterprises

## Core Module Specifications

### 1. Browser Extension
- **Platform:** Chrome (Manifest V3) + Firefox (WebExtension API)
- **Injection target:** YouTube Studio (`studio.youtube.com`) pages — Video Details, Dashboard, Comments, Analytics, Playlists
- **Implementation:** Use content scripts with Shadow DOM isolation to avoid CSS/JS conflicts with YouTube's own scripts
- **Communication:** Extension background service worker ↔ backend via authenticated REST API calls
- **UI injection points:**
  - Sidebar panel in Video Details (tag suggestions, SEO score, keyword data)
  - Overlay badges on video thumbnails in the video list
  - Floating toolbar in Comments section
  - Top bar widget in Dashboard (channel health score)

### 2. Keyword Engine
- **Data sources:**
  - YouTube Data API v3 (`search.list`, `videos.list`) for official data
  - YouTube autocomplete endpoint (`/complete/search?client=youtube&q=...`) for suggestions
  - Internal search volume index (built from aggregated autocomplete data over time)
- **Metrics to compute per keyword:**
  - Search volume (estimated monthly searches, tiered: low/medium/high/very high)
  - Competition score (0–100, based on number of results, top video performance)
  - Overall score / "TubeBuddy Score" (weighted composite)
  - Related keywords list
  - Search trend (30-day, 90-day sparkline)
- **SEO Studio scoring:**
  - Analyze title: keyword placement, length (40–70 chars optimal), click-worthiness
  - Analyze description: keyword density, length, call-to-action presence, links
  - Analyze tags: relevance, count (15–20 optimal), specificity mix (broad + niche)
  - Output: overall SEO score (0–100) + itemized recommendations

### 3. A/B Testing Engine
- **Experiment types:** Thumbnail A/B, Title A/B, or both combined
- **Rotation logic:** Switch active variant every 24 hours using YouTube Data API `videos.update`
- **Tracking:**
  - Fetch video analytics (impressions, CTR, views) via YouTube Analytics API after each rotation window
  - Store per-variant metrics in ClickHouse (high-write analytics DB)
  - Calculate statistical significance using Z-test (default threshold: p < 0.05 or 1,000 impressions per variant)
- **Result report:**
  - Per-variant CTR, impressions, and views
  - Winning variant highlighted
  - Confidence level displayed
  - One-click "Apply Winner" button

### 4. Bulk Processing Engine
- **Operations supported:** Edit title, description, tags, category, visibility, scheduled publish time, cards, end screens
- **Implementation:**
  - Job queue: BullMQ + Redis
  - Each bulk job is split into individual video update tasks (rate: max 1 YouTube API unit per second to avoid quota exhaustion)
  - Progress streamed to client via Server-Sent Events (SSE)
  - Retry logic: exponential backoff on API errors
  - Recommended batch size: ≤1,500 videos per job
- **Find & Replace:**
  - Regex-supported search across title, description, tags
  - Preview diff before applying
  - Dry-run mode

### 5. Analytics Dashboard
- **Data sources:** YouTube Analytics API (authenticated, actual user data)
- **Metrics tracked:**
  - Views, watch time, average view duration, subscriber net gain
  - CTR (impressions → clicks), impressions
  - Revenue estimates (RPM × views / 1000) — flag as estimated
  - Top-performing videos by metric
  - Keyword ranking over time per video
- **Visualization:** Line charts, bar charts, heatmaps (best publish time), sparklines

### 6. Competitor Tracker
- **Data source:** YouTube Data API v3 (public channel data only)
- **Metrics per competitor:**
  - Subscriber count history
  - Upload frequency
  - Average views per video (last 30 videos)
  - Top performing videos
  - Tag overlap score with user's channel
- **Limits:** Pro = 2 channels, Legend = 10 channels, Enterprise = unlimited

### 7. Comment Manager
- **Features:**
  - Keyword filter rules (auto-hide / auto-report matching comments)
  - Canned responses library (create, tag, search)
  - Fast-reply mode: keyboard shortcuts for common responses
  - Word cloud visualization (top words from recent comments)
  - Highlight mode: mark comments from Patreon members or recent subscribers
  - Bulk actions: approve, delete, hide, reply to filtered sets

### 8. Thumbnail Analyzer
- **Analysis via Claude Vision API (or dedicated CV model):**
  - Predicted CTR score (0–100)
  - Face detection and emotion score (faces improve CTR)
  - Text legibility score (contrast, size, readability)
  - Color contrast score
  - Mobile preview simulation (thumbnail at small sizes)
  - Comparison with top-performing thumbnails in the same niche

### 9. AI Tools (Claude API)
- **Title Generator:** Given video topic + target keyword → generate 10 title options ranked by predicted CTR and SEO score
- **Description Writer:** Given title + key points → generate SEO-optimized description with natural keyword placement
- **Tag Suggester:** Given title + description → suggest 15–20 relevant tags ordered by specificity
- **Video Idea Planner:** Given channel niche + past top performers → suggest 10 video ideas with estimated demand scores
- **All AI tools:** Include channel context (niche, tone, top 5 performing videos) in system prompt for personalization

### 10. Multi-Channel Hub (Enterprise)
- **Channel switching:** Dropdown to switch active channel context across the entire dashboard
- **Aggregate view:** Combined analytics across all channels
- **Team management:** Invite team members by email, assign roles per channel
- **Audit log:** Track all bulk actions and A/B test changes per user

- `GET /canned-responses?channel_id={id}`

## YouTube API Integration Notes

### Quota Management
- Default quota: 10,000 units/day per project
- Key operations and costs:
  - `search.list` = 100 units
  - `videos.list` = 1 unit per video
  - `videos.update` = 50 units (used for A/B rotations)
  - `channels.list` = 1 unit
  - YouTube Analytics API = separate quota, 200,000 units/day
- Strategy:
  - Cache keyword search results for 24 hours in Redis
  - Request YouTube API quota increase (up to 1M units/day) early
  - Use a per-user OAuth token for Analytics calls (does not count against app quota)
  - Implement a quota tracker per project in Redis

https://www.googleapis.com/auth/youtubepartner

## A/B Testing Engine — Implementation Detail

```typescript
// Experiment rotation job (runs via BullMQ cron, every 24 hours)
async function rotateExperimentVariant(experimentId: string) {
  const experiment = await db.experiments.findById(experimentId);
  const currentVariant = experiment.active_variant; // 'A' or 'B'
  const nextVariant = currentVariant === 'A' ? 'B' : 'A';
  const variant = experiment.variants.find(v => v.label === nextVariant);

  // Fetch current analytics before switching (close current window)
  const analytics = await youtubeAnalytics.getVideoStats(experiment.video_id);
  await db.experimentVariants.recordWindow(currentVariant, analytics);

  // Switch the video's thumbnail/title via YouTube API
  await youtube.videos.update({
    part: ['snippet'],
    requestBody: {
      id: experiment.video_id,
      snippet: {
        title: variant.title ?? undefined,
        // thumbnail set separately via thumbnails.set endpoint
      }
    }
  });

  if (variant.thumbnail_url) {
    await youtube.thumbnails.set({
      videoId: experiment.video_id,
      media: { body: await fetchThumbnailStream(variant.thumbnail_url) }
    });
  }

  await db.experiments.update(experimentId, { active_variant: nextVariant });

  // Check if experiment has reached statistical significance
  await checkSignificanceAndMaybeComplete(experimentId);
}
```

## Keyword Engine — Scoring Algorithm

```typescript
interface KeywordMetrics {
  query: string;
  searchVolume: 'very_low' | 'low' | 'medium' | 'high' | 'very_high';
  competitionScore: number;    // 0–100 (higher = more competitive)
  overallScore: number;        // 0–100 (higher = better opportunity)
  relatedKeywords: string[];
  trendData: number[];         // 30 daily relative search volume points
}

function computeKeywordScore(
  searchVolumeTier: number,   // 1–5
  competitionScore: number,   // 0–100
  channelStrengthScore: number // 0–100, user's channel authority
): number {
  // Higher volume + lower competition + higher channel strength = better score
  const volumeWeight = 0.4;
  const competitionWeight = 0.4;
  const channelWeight = 0.2;

  const volumeNorm = (searchVolumeTier / 5) * 100;
  const competitionInverted = 100 - competitionScore;

  return Math.round(
    volumeNorm * volumeWeight +
    competitionInverted * competitionWeight +
    channelStrengthScore * channelWeight
  );
}
```

## Key Technical Risks & Mitigations

| Risk | Mitigation |
|---|---|
| YouTube API quota exhaustion | Cache aggressively in Redis (keyword data: 24h TTL), use per-user tokens for Analytics calls, apply for quota increase to 1M units/day |
| YouTube Studio DOM changes breaking extension injection | Use resilient selectors (data attributes > CSS classes), add automated visual regression tests with Playwright, monitor extension error rates in Sentry |
| A/B test data contamination | Strict 24-hour rotation windows, minimum impression thresholds before declaring results, exclude videos with <200 views/day from eligibility |
| Extension store rejection (Chrome Web Store) | Avoid `unsafe-eval` in CSP, declare all permissions in manifest, provide detailed justifications in store listing |
| Bulk job failures mid-run | Idempotent job design (safe to retry), store progress in DB per video, allow resume from checkpoint |
| GDPR / data privacy | Store only public YouTube data for competitor tracking, encrypt OAuth tokens at rest (AES-256), provide data deletion flow |
